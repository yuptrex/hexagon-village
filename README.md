# Hexagon Village — Walkable 3D Viewer

A browser-based first-person walkthrough of a village built from the
**KayKit Medieval Hexagon Pack** glTF assets, rendered with [Three.js](https://threejs.org/).
No build step, no server code — just static files, deployable straight to **GitHub Pages**.

## Controls

**Desktop**
- `W A S D` or arrow keys — move
- Mouse — look around (click once to lock the pointer)
- `Shift` — run
- `Space` — jump
- `Esc` — release the mouse

**Mobile**
- Left-side joystick — move
- Drag anywhere on the right side of the screen — look around (full 360°)
- Bottom-right button — jump

## Deploying to GitHub Pages

1. **Create a new repository** on GitHub (e.g. `hexagon-village`). Public repos get free Pages hosting; private repos need a paid plan for Pages.

2. **Upload these files**, keeping the folder structure exactly as-is:
   ```
   index.html
   main.js
   village.json
   .nojekyll
   assets/
     tiles/base/...
     tiles/roads/...
     buildings/blue/...
     buildings/red/...
     buildings/neutral/...
     decoration/nature/...
     decoration/props/...
   ```
   Easiest path — in the GitHub web UI: **Add file → Upload files**, then drag the whole `village-site` folder contents in (GitHub preserves subfolders when you drag a folder in Chrome/Edge).

   Or via git from your machine:
   ```bash
   git clone https://github.com/<your-username>/hexagon-village.git
   cd hexagon-village
   # copy all files from this village-site folder in here
   git add .
   git commit -m "Add walkable hexagon village"
   git push
   ```

3. **Turn on Pages**: repo → **Settings → Pages** → under "Build and deployment", set **Source** to `Deploy from a branch`, branch `main`, folder `/ (root)`. Save.

4. Wait about a minute, then your site is live at:
   ```
   https://<your-username>.github.io/hexagon-village/
   ```

That's it — everything runs client-side. Three.js itself loads from a CDN (`unpkg.com`) via an import map in `index.html`, so you don't need to upload the library.

## Customizing the layout

`village.json` describes every tile and object placed in the world (position, rotation, which `.gltf` file). It was generated by a small script from the pack's hex-grid spacing. To rearrange buildings, add more houses, or grow the village, just edit the `tiles` / `objects` arrays — each entry is:

```json
{ "file": "buildings/blue/building_castle_blue.gltf", "x": 0, "y": 0, "z": 0, "ry": 0 }
```

`file` is the path under `assets/`, `x`/`z` are world position, `ry` is Y-axis rotation in degrees.

## Notes

- Total asset payload is about 7 MB — loads in a few seconds on a normal connection; a progress bar shows while models stream in.
- Collision is a simple radius check around buildings/trees so you can't walk through them, not full mesh physics.
